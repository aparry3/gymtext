# Context System & Template Engine

## Overview
The context system resolves dynamic data at agent invocation time and injects it into prompts. It works alongside the template engine to build fully-resolved prompts from database-stored templates.

## Template Engine

Agent definitions store a `user_prompt_template` field with `{{variable}}` placeholders. At invocation time, the SimpleAgentRunner resolves these from the `params` object:

```typescript
// Template in DB:
"Generate a workout for {{userName}} based on their {{experienceLevel}} level."

// Params passed to invoke():
{ userName: "Sarah", experienceLevel: "intermediate" }

// Resolved prompt:
"Generate a workout for Sarah based on their intermediate level."
```

The template engine is simple string substitution — no conditionals or loops. All logic lives in the context providers that prepare the data.

## Context Strings

The `context` field in `SimpleAgentInvokeParams` accepts an array of pre-resolved context strings. These are injected as user-role messages with section: "context" in the message array, appearing between the system prompt and conversation history.

Services prepare context before calling `agentRunner.invoke()`:

```typescript
const result = await agentRunner.invoke('workout:format', {
  input: workoutData,
  params: { user, timezone },
  context: [
    formatFitnessProfile(user.fitnessProfile),
    formatCurrentMicrocycle(microcycle),
  ],
});
```

## Key Context Patterns

### User Profile Context
Fitness profile formatted as markdown — includes goals, experience level, available equipment, training preferences, injuries/limitations. Generated by `formatters/fitnessProfile.ts`.

### Fitness Plan Context
Current mesocycle structure — phases, focus areas, progression scheme. Formatted by services before agent invocation.

### Current Week/Microcycle Context
Weekly training pattern — which days have which workouts, rest days, volume distribution. Formatted by `formatters/microcycle.ts`.

### Workout Details Context
Specific workout structure — blocks, exercises, sets, reps, intensity. Used when modifying existing workouts.

### Date/Timezone Context
Current date and user's timezone — critical for "today's workout" resolution. Uses Luxon-based date utilities.

## How Context Flows

1. Service prepares to invoke an agent
2. Service calls domain services/repositories to fetch relevant data
3. Service formats data into context strings using formatters
4. Service passes context strings to `agentRunner.invoke()` via `context` param
5. AgentRunner injects context strings as user-role messages after system prompt
6. LLM sees: system prompt → context → examples → history → user prompt

## Message Assembly with Context

```
┌─────────────────────────┐
│ System Prompt            │  ← From agent_definitions.system_prompt
├─────────────────────────┤
│ Context Message 1        │  ← context[0] (e.g., fitness profile)
│ Context Message 2        │  ← context[1] (e.g., current week)
├─────────────────────────┤
│ Example User Message     │  ← From agent_definitions.examples
│ Example Assistant Reply  │
├─────────────────────────┤
│ Previous User Message    │  ← Conversation history
│ Previous Assistant Reply │
├─────────────────────────┤
│ Current User Prompt      │  ← Resolved from user_prompt_template
└─────────────────────────┘
```
